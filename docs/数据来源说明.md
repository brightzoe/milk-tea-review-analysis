# 数据来源说明

## 1. 数据组成

本项目的 `data/reviews.csv` 由两部分组成：

1. ASAP 公开中文餐馆评论 sample 中抽取的真实餐饮评论片段。
2. 人工整理的奶茶消费场景样例。

当前最终数据集规模为 **500 条**。其中约 32 条来自 ASAP sample 中经过饮品场景过滤后的真实奶茶/饮品评论片段，约 468 条为人工整理的奶茶消费场景样例。这样既能引入部分真实评论表达，也能覆盖奶茶项目需要分析的甜度、冰量、小料、奶盖、外卖包装等专属问题。

## 2. ASAP 数据

ASAP 是美团点评公开的中文餐馆评论数据集，仓库地址：

```text
https://github.com/Meituan-Dianping/asap
```

本项目使用其中 sample CSV 文件：

```text
data/external/asap_train_sample.csv
data/external/asap_dev_sample.csv
data/external/asap_test_sample.csv
```

ASAP 原始评论来自餐馆场景，覆盖口味、服务、价格、等待、环境等消费体验维度。虽然它不是专门的奶茶评论数据，但这些维度与奶茶店消费体验有较高重合度，因此适合抽取部分评论作为真实语料补充。

## 3. 抽取方式

数据构建脚本为：

```text
scripts/build_dataset.py
```

脚本处理流程：

1. 读取 `data/external/` 下的 ASAP sample 文件。
2. 从 `reviewbody` 中按句子切分评论正文。
3. 优先选择包含口味、服务、价格、等待、环境、推荐等消费体验关键词的句子。
4. 将过长句子截取为适合展示和分类的片段。
5. 根据 `star` 字段映射情感标签。
6. 补充项目需要的 `brand`、`product`、`channel` 字段。
7. 与人工整理的奶茶消费场景样例混合，生成最终 `data/reviews.csv`。

## 4. 字段映射

| 本项目字段 | 来源或处理方式 |
|---|---|
| `content` | ASAP `reviewbody` 中抽取的餐饮体验句子，或人工整理的奶茶场景样例 |
| `rating` | ASAP 按 `star` 映射；奶茶样例按情感标签映射 |
| `label` | `star >= 4` 为正面，`star <= 2` 为负面，其余为中性 |
| `brand` | 映射为课程项目样例品牌：蜜雪茶姬、瑞巴克、爷爷喜欢茶 |
| `product` | 映射为奶茶类产品，如珍珠奶茶、柠檬茶、奶盖茶等 |
| `channel` | ASAP 评论默认映射为门店渠道，奶茶样例包含门店和外卖 |

## 5. 奶茶场景样例

ASAP 是餐馆评论数据，其中并不大量包含奶茶专属表达。因此本项目保留人工整理的奶茶场景样例，用于覆盖以下问题：

- 口味：太甜、太淡、苦、腻、难喝、茶味香、清爽。
- 小料：珍珠硬、椰果、布丁、红豆、芋圆、燕麦、仙草。
- 冰量和糖度：三分糖、五分糖、七分糖、全糖、少冰、去冰。
- 价格：贵、便宜、划算、涨价、用券、性价比。
- 出餐：排队、等很久、出餐慢、出餐快。
- 服务：态度好、态度差、不耐烦、贴心。
- 包装配送：漏、洒、杯盖不紧、配送慢、送错。
- 产品：料少、缺料、冰太多、奶盖细腻、奶盖太腻。

## 6. 构建命令

如果需要重新生成数据，可以运行：

```bash
python scripts/build_dataset.py
```

重新生成数据后，再运行主程序刷新输出：

```bash
python -m src.main
```

## 7. 隐私与合规说明

- 本项目仅使用 ASAP 公开仓库中的 sample 文件。
- 不保留用户昵称、联系方式等个人信息。
- `brand` 字段使用课程项目样例品牌，避免误用真实商家名称。
- 最终数据只用于课程项目中的情感分析、问题归因和系统展示。
