# 总体设计

## 1. 系统定位

本系统面向奶茶店评论数据，完成评论读取、文本预处理、情感分析、问题归因、品牌对比、图表生成和报告导出。系统采用命令行方式运行，不依赖爬虫和在线接口，适合作为课程项目进行展示和测试。

系统同时保留两类分析方法：

- 规则模型：使用情感词典、否定词、程度副词和问题类别词典，结果可解释。
- 机器学习模型：使用 `TF-IDF + LogisticRegression`，展示监督学习分类流程。

## 2. 总体架构

系统分为四层：

| 层次 | 主要模块 | 职责 |
|---|---|---|
| 数据层 | `ReviewReader`、`Review` | 读取 CSV，校验字段，封装评论对象 |
| 文本处理层 | `TextCleaner`、`SentimentLexicon` | 文本清洗、关键词提取、词典管理 |
| 分析层 | `SentimentAnalyzer`、`ProblemAnalyzer`、`SklearnSentimentClassifier`、`StatisticsManager` | 情感判断、问题归因、机器学习分类、统计汇总 |
| 展示层 | `ChartGenerator`、`ReportExporter`、`SystemController` | 图表、CSV、Markdown 报告和流程控制 |

## 3. 系统流程

```text
读取 reviews.csv
      ↓
校验字段和标签
      ↓
生成 Review 对象列表
      ↓
文本清洗与关键词提取
      ↓
规则情感分析
      ↓
非正面评论问题归因
      ↓
sklearn 模型训练与评估
      ↓
机器学习模型预测每条评论
      ↓
统计品牌、情感、关键词、问题类别
      ↓
生成改进建议
      ↓
生成图表、CSV、Markdown 和模型评估文件
```

## 4. 模块划分

| 模块 | 输入 | 输出 |
|---|---|---|
| `ReviewReader` | `data/reviews.csv` | `list[Review]` |
| `TextCleaner` | 评论正文 | 清洗后的关键词列表 |
| `SentimentAnalyzer` | `Review.tokens` | 规则情感得分和情感标签 |
| `ProblemAnalyzer` | 评论正文、规则情感标签 | 问题类别列表 |
| `SklearnSentimentClassifier` | 评论正文和人工标签 | 评估结果、机器学习预测标签 |
| `StatisticsManager` | 完成分析的评论列表 | `AnalysisSummary` |
| `SuggestionGenerator` | 品牌问题统计 | 品牌改进建议 |
| `ChartGenerator` | `AnalysisSummary` | 三张 PNG 图表 |
| `ReportExporter` | `AnalysisSummary`、评论列表 | CSV 和 Markdown 报告 |

## 5. 数据流设计

评论数据只在入口读取一次。之后系统通过 `Review` 对象在各模块之间传递结果，避免多模块重复解析 CSV。

`Review` 对象会逐步补充以下分析字段：

- `tokens`
- `rule_sentiment_score`
- `rule_sentiment_label`
- `ml_sentiment_label`
- `problem_categories`

整体统计结果使用 `AnalysisSummary` 保存，品牌统计使用 `BrandReport` 保存。

## 6. 输出设计

系统运行后输出：

| 文件 | 内容 |
|---|---|
| `output/analyzed_reviews.csv` | 每条评论的规则模型结果、机器学习结果和问题归因 |
| `output/analysis_report.md` | 综合分析报告 |
| `output/ml_evaluation.txt` | sklearn 分类器评估结果 |
| `output/sentiment_model.pkl` | 持久化的 sklearn 分类模型 |
| `output/charts/sentiment_distribution.png` | 总体情感分布 |
| `output/charts/brand_sentiment_compare.png` | 品牌情感对比 |
| `output/charts/problem_ranking.png` | 问题类别排行 |

## 7. 设计特点

- 类职责清晰，读取、清洗、分析、统计、导出分离。
- 数据结构使用明确，便于解释课程要求中的数据结构应用。
- 规则模型和机器学习模型并存，兼顾可解释性和分类能力。
- 输出文件完整，便于提交和展示。
