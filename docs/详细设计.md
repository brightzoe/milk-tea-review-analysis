# 详细设计

## 1. 核心实体设计

### 1.1 Review

`Review` 表示单条评论，是系统中最核心的数据对象。

主要字段：

| 字段 | 含义 |
|---|---|
| `review_id` | 评论编号 |
| `date` | 评论日期 |
| `brand` | 品牌名称 |
| `product` | 饮品名称 |
| `channel` | 评论来源，门店或外卖 |
| `content` | 评论正文 |
| `rating` | 用户评分 |
| `label` | 人工标注情感类别 |
| `tokens` | 清洗后的关键词 |
| `rating_sentiment_label` | 由评分映射得到的情感类别 |
| `rule_sentiment_score` | 规则模型情感得分 |
| `rule_sentiment_label` | 规则模型情感类别 |
| `ml_sentiment_label` | 机器学习模型预测类别 |
| `problem_categories` | 问题归因结果 |

### 1.2 BrandReport

`BrandReport` 保存单个品牌的统计结果，包括评论数量、情感数量、平均得分、主要问题、高频关键词和改进建议。

### 1.3 AnalysisSummary

`AnalysisSummary` 保存整体分析结果，用于生成报告和图表。

## 2. 类职责设计

| 类名 | 职责 |
|---|---|
| `ReviewReader` | 读取 CSV、校验字段、生成 `Review` 对象 |
| `TextCleaner` | 清洗文本、提取关键词、过滤停用词 |
| `SentimentLexicon` | 管理正负向词、程度副词和否定词 |
| `SentimentAnalyzer` | 根据规则词典计算情感分数和类别 |
| `ProblemAnalyzer` | 根据问题词典识别评论问题类别 |
| `SklearnSentimentClassifier` | 训练、预测和评估机器学习分类器 |
| `StatisticsManager` | 统计情感分布、问题排行、品牌结果和一致率 |
| `SuggestionGenerator` | 根据问题类别生成改进建议 |
| `ChartGenerator` | 生成情感分布、品牌对比、问题排行图 |
| `ReportExporter` | 导出 CSV 和 Markdown 报告 |
| `SystemController` | 串联完整业务流程 |

## 3. 数据结构设计

| 数据结构 | 使用位置 | 设计理由 |
|---|---|---|
| `list[Review]` | 评论列表 | 保持输入顺序，便于批量分析 |
| `dict[str, float]` | 情感词典 | 按词快速查找情感分数 |
| `dict[str, list[str]]` | 问题词典 | 将问题类别映射到关键词列表 |
| `set[str]` | 否定词、停用词 | 去重并提高成员判断效率 |
| `Counter` | 统计关键词和问题 | 简化频次统计 |
| `dataclass` | 评论和统计实体 | 结构清晰，减少样板代码 |

## 4. 数据读取设计

输入文件固定为 CSV，字段为：

```csv
review_id,date,brand,product,channel,content,rating,label
```

校验规则：

- 字段必须完整。
- `rating` 必须是 1 到 5 的整数。
- `label` 必须是 `正面`、`中性`、`负面`。
- 评论正文不能为空。

发现非法数据时抛出 `ValueError`，避免错误数据进入后续分析流程。

当前数据构建方式：

- 从 ASAP 公开中文餐馆评论 sample 中通过饮品场景白名单/黑名单过滤，仅保留与奶茶、果茶、咖啡、饮料等相关的评论片段。
- 过滤规则：片段需命中奶茶/饮品关键词；若同时命中烧烤、火锅、海鲜、日料、西餐等餐饮类型黑名单，则丢弃。
- 将筛选后的真实评论片段映射到项目字段 `brand`、`product`、`channel`、`rating`、`label`。
- 使用人工扩写的奶茶场景样例补足数据量，最终由 `scripts/build_dataset.py` 生成 `data/reviews.csv`。

## 5. 文本预处理设计

`TextCleaner` 负责：

- 去除常见中文和英文标点。
- 合并多余空格。
- 根据词典进行短语匹配。
- 过滤停用词。
- 在分析单条评论时，先移除当前评论的品牌名和产品名，避免品牌或产品名称中的情感词干扰规则模型。

关键词匹配采用“长度优先、字典序稳定”的排序方式：

```text
先匹配长短语，再匹配短词；同长度词按固定顺序处理。
```

这样可以避免 `态度差`、`等很久`、`包装漏` 等短语被拆散，也保证多次运行结果稳定。

例如，品牌名 `爷爷喜欢茶` 中包含 `喜欢`，如果直接参与情感词匹配，会把中性或负面评论错误地拉高为正面。系统在 `SystemController` 中调用 `extract_tokens_for_review()`，传入当前 `brand` 和 `product` 作为忽略项，只对评论正文中的真实评价词进行情感评分。

## 6. 规则情感分析设计

规则模型使用四类词典：

- 正面词：如 `好喝`、`实惠`、`满意`。
- 负面词：如 `太甜`、`贵`、`等很久`。
- 程度副词：如 `非常`、`很`、`有点`。
- 否定词：如 `不`、`没`、`没有`。

评分逻辑：

```text
遍历 tokens：
  如果 token 是情感词，取得基础分
  如果前一个 token 是程度副词，乘以程度权重
  如果前一个或前两个 token 是否定词，当前分数取反
  累加到总分
```

分类规则：

```text
score >= 1：正面
-1 < score < 1：中性
score <= -1：负面
```

该算法的优点是可解释，能说明评论为什么被判为某类情感。

由于奶茶评论普遍较短，单条评论中常常只有一个明确情感词，因此正面阈值设置为 `1`。这样可以减少“好喝”“细腻”“稳定”等短评价被误判为中性的情况。

## 7. 机器学习分类设计

机器学习模型采用 `scikit-learn`：

```text
TfidfVectorizer(analyzer="char", ngram_range=(1, 2))
LogisticRegression(max_iter=1000)
```

选择字符级 n-gram 的原因：

- 不依赖中文分词库。
- 适合短评论。
- 能学习常见字词组合与情感标签之间的关系。

训练与评估：

- 使用 `label` 字段作为监督标签。
- `rating` 只作为辅助参考字段，不直接作为文本情感分析结果。
- 采用 8:2 划分训练集和测试集。
- 固定 `random_state=42`。
- 输出 accuracy、precision、recall、f1-score、classification report 和 confusion matrix。

## 8. 评分映射与一致率设计

系统会将用户评分映射为 `rating_sentiment_label`：

```text
rating >= 4：正面
rating == 3：中性
rating <= 2：负面
```

该字段用于和人工标签、规则模型结果、机器学习模型结果进行对比。系统最终分析不直接使用 `rating` 判断情感，而是根据评论文本进行规则分析和机器学习预测，再用 `rating_sentiment_label` 作为辅助参考。

`StatisticsManager` 会统计三类一致率：

- 规则模型与人工标签一致率。
- 机器学习模型与人工标签一致率。
- 评分映射与人工标签一致率。

## 9. 问题归因设计

系统维护七类问题：

- 口味问题
- 价格问题
- 出餐问题
- 服务问题
- 包装问题
- 配送问题
- 产品问题

只对非正面评论进行问题归因。一条评论可同时命中多个类别，例如：

```text
评论：太甜了，包装还漏了，等了很久。
归因：口味问题、包装问题、出餐问题
```

## 10. 建议生成设计

`SuggestionGenerator` 根据品牌的 Top 3 问题类别生成建议。

示例：

- 出餐问题较多：建议高峰期提前备料，优化点单和取餐流程。
- 包装问题较多：建议检查杯盖密封性，加强外卖二次封装。
- 产品问题较多：建议检查小料新鲜度和添加标准。

## 11. 异常与边界处理

- CSV 缺字段：抛出 `ValueError`。
- 非法评分：抛出 `ValueError`。
- 非法标签：抛出 `ValueError`。
- 图表字体缺失：使用候选字体，图表生成失败时打印提示，不影响 CSV 和 Markdown 输出。
- 某类样本过少：机器学习模块降级为普通随机划分。

## 12. 输出文件设计

- CSV 保留逐条评论分析结果，适合检查程序正确性。
- Markdown 报告保留总体统计、品牌对比、问题排行、模型评估和改进建议。
- `sentiment_model.pkl` 保存训练好的 sklearn 模型，便于后续复用或部署。
- PNG 图表用于系统运行实例展示。
