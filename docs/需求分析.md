# 需求分析

## 项目背景

奶茶门店的评论中包含大量消费体验信息，例如口味、价格、排队、包装、配送和服务态度。人工逐条查看评论效率较低，也难以及时发现集中出现的问题。

本系统面向奶茶店评论数据，自动完成情感分析、问题归因和服务建议生成，帮助快速了解不同门店或品牌的用户反馈。

当前样例数据结合了 ASAP 公开中文餐馆评论 sample 和人工整理的奶茶消费场景样例。这样既能减少评论表达过于规整的问题，也能保留奶茶项目需要分析的甜度、冰量、小料、包装等专属问题。

## 用户需求

- 能导入本地评论数据。
- 能看到评论整体情感倾向。
- 能比较不同品牌的满意度。
- 能知道负面评论主要集中在哪些问题。
- 能导出报告用于课程展示。

## 功能需求

- 数据读取：读取 CSV 文件并校验字段。
- 文本处理：清洗评论文本并提取关键词。
- 情感分析：同时提供规则模型和 sklearn 分类器。
- 问题归因：识别口味、价格、出餐、服务、包装、配送、产品问题。
- 结果展示：导出 CSV、Markdown 报告和图表。

## 非功能需求

- 结构清晰，体现面向对象设计。
- 结果可解释，便于在报告中说明算法过程。
- 不依赖爬虫和网络接口。
- 能通过单元测试验证主要功能。
