# 运行实例与更新计划

## 1. 运行命令

```bash
python -m src.main
```

系统默认读取：

```text
data/reviews.csv
```

默认输出：

```text
output/
```

## 2. 当前数据规模

当前样例数据共 500 条，包含三个课程项目样例品牌：

- 蜜雪茶姬
- 瑞巴克
- 爷爷喜欢茶

数据包含正面、中性、负面三类人工标签，可用于机器学习模型训练和评估。

数据来源由两部分组成：

- ASAP 公开中文餐馆评论 sample 中经过饮品场景过滤后抽取的真实奶茶/饮品评论片段，约 32 条。
- 人工整理的奶茶消费场景样例，约 468 条，用于覆盖珍珠、小料、甜度、冰量、奶盖、外卖包装等奶茶专属问题。

（ ASAP 原始 sample 覆盖烧烤、火锅、海鲜、日料等多种餐饮场景，本项目通过白名单/黑名单过滤后仅保留奶茶/饮品相关片段，因此数量较少。）

## 5. 用户反馈与设想

在课程展示前，我让两位同学试用了本系统，并收集了以下具体反馈：

- **负面评论样例不够直观**：有同学反馈“只看统计数字不知道评论具体说了什么”。因此在 `analysis_report.md` 中增加了“典型负面评论样例”表格，并在问题归因中列出具体命中关键词。
- **品牌名干扰情感判断**：调试时发现品牌名“爷爷喜欢茶”中包含“喜欢”，会让中性评论被误判为正面。已通过在 `TextCleaner` 中忽略当前评论的品牌名和产品名解决，并补充了回归测试。
- **图表字体在不同电脑显示不一致**：在室友电脑上运行时，matplotlib 找不到 SimHei 字体。已设置多个字体候选，并保证即使图表渲染异常，CSV 和 Markdown 报告仍能正常生成。
- **希望保存训练好的模型**：有同学问能不能把 sklearn 模型保存下来下次直接用。已在 `SklearnSentimentClassifier` 中增加 `save_model/load_model` 方法，运行主程序后会输出 `sentiment_model.pkl`。

除以上实际反馈外，潜在用户可能还会关注：

- 是否能快速看到负面评论集中原因。
- 是否能比较不同品牌的满意度。
- 是否能导出结果用于课程报告。
- 情感分类是否能解释原因。

本系统通过规则模型解释具体问题，通过机器学习模型展示自动分类能力，两者互相补充。
