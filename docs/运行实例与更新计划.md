# 运行实例与更新计划

## 1. 运行命令

```bash
python -m src.main
```

系统默认读取：

```text
data/reviews.csv
```

默认输出：

```text
output/
```

## 2. 当前数据规模

当前样例数据共 500 条，包含三个课程项目样例品牌：

- 蜜雪茶姬
- 瑞巴克
- 爷爷喜欢茶

数据包含正面、中性、负面三类人工标签，可用于机器学习模型训练和评估。

数据来源由两部分组成：

- ASAP 公开中文餐馆评论 sample 中经过饮品场景过滤后抽取的真实奶茶/饮品评论片段，约 36 条。
- 人工整理的奶茶消费场景样例，约 464 条，用于覆盖珍珠、小料、甜度、冰量、奶盖、外卖包装等奶茶专属问题。

（ ASAP 原始 sample 覆盖烧烤、火锅、海鲜、日料等多种餐饮场景，本项目通过白名单/黑名单过滤后仅保留奶茶/饮品相关片段，因此数量较少。）

## 3. 运行结果文件

| 文件 | 说明 |
|---|---|
| `output/analyzed_reviews.csv` | 每条评论的分析结果 |
| `output/analysis_report.md` | 综合分析报告 |
| `output/ml_evaluation.txt` | 机器学习模型评估 |
| `output/charts/sentiment_distribution.png` | 总体情感分布图 |
| `output/charts/brand_sentiment_compare.png` | 品牌情感对比图 |
| `output/charts/problem_ranking.png` | 问题类别排行图 |

## 4. 示例分析结果

示例评论：

```text
太甜了，包装还漏了，等了很久。
```

系统可以输出：

```text
规则情感类别：负面
问题归因：口味问题、包装问题、出餐问题
```

## 5. 用户反馈设想

试用者可能关注：

- 是否能快速看到负面评论集中原因。
- 是否能比较不同品牌的满意度。
- 是否能导出结果用于课程报告。
- 情感分类是否能解释原因。

本系统通过规则模型解释具体问题，通过机器学习模型展示自动分类能力，两者互相补充。

## 6. 后续更新计划
